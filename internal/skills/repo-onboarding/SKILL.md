# repo-onboarding

## Workflow

1. Build a directory-level map (top-level, core modules, test directories).
2. Confirm startup, build, and dependency management commands.
3. Outline the runtime path (entrypoint to core modules to outputs/side effects).
4. List high-value next reads and commands.

## Output Contract

- Repo Map
- Build and Test Entry
- Core Runtime Flow
- Next Read and Run Checklist
