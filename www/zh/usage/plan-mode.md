# Plan 模式

Plan 模式下，Agent 会先输出一个逐步执行计划的文字描述，由你审阅并确认后再开始执行。这让你对大型任务或高风险变更有完整的可见性和控制权。

## 如何激活

在已有会话中输入：

```text
/plan
```

随时切回 Build 模式：

```text
/build
```

## 工作流程

1. 你描述任务
2. Agent 输出有序的执行步骤列表，包含预估范围
3. 你审阅计划，可提问或要求修订
4. 确认后 Agent 逐步执行
5. 每个步骤完成后可验证再推进

## 适用场景

| 场景                     | Plan 模式的价値                |
| ------------------------ | ------------------------------ |
| 跨多个包的大规模重构     | 动居任何文件之前先看到全局影响 |
| 有序依赖的功能实现       | 防止变更顺序错乱               |
| 数据库迁移或 Schema 变更 | 运行前验证迁移顺序             |
| 快速熟悉复杂新仓库       | 明确范围后再提交变更           |

:::tip
简单独立的任务用 Build 模式更快。希望在执行前**看到并确认执行方案**时再切换到 Plan 模式。
:::

## 示例会话

```text
/plan
重构认证模块：将 token 校验逻辑抽取到独立包，更新所有调用方，并为新包补充单元测试。
```

Agent 会回复类似这样的计划：

```
执行计划：
1. 读取当前认证模块结构（internal/auth/）
2. 定位所有 token 校验调用位置
3. 创建新包 internal/tokenval/并迁入相关逻辑
4. 更新 internal/auth/ 和 internal/api/ 中的调用方
5. 为 internal/tokenval/ 编写单元测试
6. 运行完整测试套件确认无回幒
```

你审阅后确认开始执行。

## 执行过程中的控制

Plan 模式执行期间你可以：

- **要求 Agent 修订**某个具体步骤
- **在任意步骤后暫停**检查结果
- **切回 Build 模式**（`/build`）让 Agent 自由继续
- 对长计划**提高 `max_iterations`** 防止预算耗尽
